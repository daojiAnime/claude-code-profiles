# 提示词模板库

本目录包含常用的提示词模板，帮助快速构建高质量的提示词。

## 模板列表

### 1. basic_task.xml - 基础任务模板
适用于一般性任务，包含任务目标、上下文、要求等基本元素。

**使用场景:**
- 简单的创作任务
- 数据处理请求
- 一般性问答

### 2. with_examples.xml - 示例驱动模板
使用多个示例来说明期望的输出模式。

**使用场景:**
- 格式转换
- 分类任务
- 风格模仿

### 3. chain_of_thought.xml - 思维链模板
引导 Claude 展示推理过程，提高复杂问题的准确性。

**使用场景:**
- 数学问题
- 逻辑推理
- 复杂决策
- 代码调试

### 4. role_based.xml - 角色扮演模板
为 Claude 分配特定角色和专业视角。

**使用场景:**
- 专业咨询
- 技术评审
- 教学辅导

### 5. analysis.xml - 分析模板
用于多维度、结构化的分析任务。

**使用场景:**
- 产品评估
- 竞品分析
- 技术选型
- 风险评估

### 6. code_generation.xml - 代码生成模板
专门用于代码生成任务，包含功能、技术和质量要求。

**使用场景:**
- 函数实现
- 工具脚本
- API 开发
- 算法实现

## 使用方法

1. 选择合适的模板
2. 复制模板内容
3. 填充 `[占位符]` 部分
4. 根据具体需求调整结构
5. 发送给 Claude

## 模板组合

可以组合多个模板的特性：

**示例：角色 + 思维链**
```xml
<role>
  你是一位资深架构师
</role>

<thinking_framework>
  请一步步分析这个系统设计方案
</thinking_framework>
```

**示例：示例 + 角色**
```xml
<role>
  你是一位文案编辑
</role>

<examples>
  <!-- 展示优秀文案示例 -->
</examples>
```

## 最佳实践

1. **选择合适的模板** - 根据任务类型选择最匹配的模板
2. **填充完整信息** - 不要留空关键字段
3. **调整而非照搬** - 根据实际需求调整模板结构
4. **测试和迭代** - 观察输出质量并优化提示词

## 自定义模板

可以基于这些模板创建自己的模板：

1. 识别常见任务模式
2. 提取可重用的结构
3. 创建模板文件
4. 记录使用场景

## 贡献

欢迎添加新的模板！好的模板应该：
- 解决特定类型的任务
- 结构清晰、易于填充
- 包含足够的指导
- 经过实践验证
