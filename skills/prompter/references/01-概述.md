# 提示工程概述

## 什么是提示工程？

提示工程是设计和优化输入提示词的过程，以引导 AI 模型产生更准确、相关和有用的输出。对于 Claude 这样的大语言模型，提示词的质量直接影响响应的质量。

## 核心原则

### 1. 清晰性优先
- 使用简洁、直接的语言
- 避免歧义和模糊表达
- 明确说明期望的输出格式

### 2. 提供上下文
- 给 Claude 足够的背景信息
- 说明任务的目标和约束
- 提供相关的领域知识

### 3. 结构化组织
- 使用 XML 标签分隔不同部分
- 采用清晰的层次结构
- 保持逻辑流畅

### 4. 迭代优化
- 测试并观察输出结果
- 根据反馈调整提示词
- 记录有效的模式

## 提示词的基本结构

```xml
<任务描述>
[清晰说明要完成的任务]
</任务描述>

<上下文>
[提供必要的背景信息]
</上下文>

<要求>
[列出具体要求和约束]
</要求>

<输出格式>
[说明期望的输出格式]
</输出格式>
```

## 常见应用场景

1. **内容生成** - 创作文章、代码、创意文本
2. **分析总结** - 提取关键信息、生成摘要
3. **问答对话** - 回答问题、提供建议
4. **转换翻译** - 格式转换、语言翻译
5. **推理决策** - 逻辑分析、问题解决

## 渐进式优化路径

1. **基础级** - 保持清晰和直接
2. **进阶级** - 使用示例和 XML 标签
3. **高级级** - 应用思维链和角色分配
4. **专家级** - 链式提示和扩展思考

## 最佳实践总结

- ✅ 明确任务目标
- ✅ 提供充分上下文
- ✅ 使用结构化格式
- ✅ 给出具体示例
- ✅ 迭代测试优化
- ❌ 避免模糊指令
- ❌ 避免过度复杂
- ❌ 避免假设背景知识
