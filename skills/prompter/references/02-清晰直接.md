# 保持清晰、直接和详细

## 核心原则

Claude 的性能与提示词的清晰度直接相关。越清晰、越具体的指令，产生的结果越好。

## 清晰性的三个维度

### 1. 明确性 (Clarity)
- 使用简单、直接的语言
- 避免行话和模糊术语
- 一次只表达一个想法

**❌ 模糊:**
```
帮我写点东西
```

**✅ 明确:**
```
请写一篇 500 字的博客文章，介绍 Python 列表推导式的基本用法，
包含 3 个实际示例，面向初学者。
```

### 2. 具体性 (Specificity)
- 提供具体的细节和参数
- 说明期望的长度、格式、风格
- 列出必须包含的要点

**❌ 笼统:**
```
分析这个数据集
```

**✅ 具体:**
```
分析附件中的销售数据集（sales_2024.csv），重点关注：
1. 月度销售趋势
2. Top 5 产品类别
3. 地区分布差异
以 markdown 表格形式输出结果。
```

### 3. 详细性 (Detail)
- 提供必要的背景信息
- 说明任务的目的和用途
- 包含相关的约束条件

**❌ 简略:**
```
翻译这段文字
```

**✅ 详细:**
```
将下面的产品描述从中文翻译成英文：
- 目标受众：北美市场的年轻消费者
- 语气：轻松、友好、专业
- 保留品牌名称不翻译
- 控制在 200 词以内

[产品描述文本]
```

## 实用技巧

### 使用"金字塔原则"

先说结论，再说细节：

```xml
<任务>
创建一个 Python 函数来计算斐波那契数列

<要求>
- 函数名: fibonacci
- 参数: n (数列长度)
- 返回: 列表形式的数列
- 添加类型提示
- 包含 docstring
- 处理边界情况 (n <= 0)
</要求>
</任务>
```

### 分解复杂任务

将大任务拆分成小步骤：

```
请按以下步骤分析用户反馈：

步骤 1: 统计总共收到多少条反馈
步骤 2: 按情感分类（正面/负面/中性）
步骤 3: 提取最常见的 5 个主题
步骤 4: 为每个主题提供代表性引用
步骤 5: 总结关键发现和建议
```

### 提供输出示例

展示期望的输出格式：

```
请将以下信息转换为 JSON 格式：

输入: "张三，男，28岁，软件工程师"

期望输出:
{
  "name": "张三",
  "gender": "男",
  "age": 28,
  "occupation": "软件工程师"
}
```

## 常见陷阱

### ❌ 陷阱 1: 假设隐含知识

```
用那个库来处理图片
```
→ Claude 不知道"那个库"是什么

### ❌ 陷阱 2: 多重解释

```
快速写个脚本
```
→ "快速"可能指执行速度快，也可能指快速完成编写

### ❌ 陷阱 3: 缺少约束

```
生成测试数据
```
→ 没有说明数据量、格式、字段

## 检查清单

在发送提示词前，检查是否：

- [ ] 任务目标清晰明确
- [ ] 提供了必要的上下文
- [ ] 说明了输出格式
- [ ] 包含了具体要求
- [ ] 定义了成功标准
- [ ] 考虑了边界情况

## 实践练习

**原始提示词:**
```
帮我写个程序
```

**优化后:**
```
创建一个 Python 命令行程序，功能如下：

功能: 批量重命名文件
- 输入: 文件夹路径
- 操作: 将所有图片文件重命名为 "IMG_序号.扩展名"
- 序号: 从 001 开始，补齐 3 位
- 支持格式: .jpg, .png, .gif
- 添加确认提示，避免误操作

技术要求:
- 使用 pathlib 处理路径
- 添加错误处理
- 提供使用帮助文档
```

## 总结

清晰、直接、详细的提示词是高质量输出的基础。投入时间明确需求，会大大提升 Claude 的表现。
